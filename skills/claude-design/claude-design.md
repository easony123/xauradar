# Claude Design

Design-first workflow for visual output using HTML/CSS/JS as a prototyping medium.

## Use When

- Building interactive prototypes or UI mockups
- Exploring multiple design directions
- Creating presentation-style HTML slides
- Creating motion/animation concept demos
- Reviewing visual quality and proposing fixes

## Core Rules

1. Confirm context first:
- Ask for existing brand assets, screenshots, or design systems
- Do not design from scratch if reusable context exists

2. Define a visual system before coding:
- Type scale and font pairing
- Color roles and contrast
- Spacing rhythm and layout logic

3. Show early, iterate fast:
- Share a rough first pass quickly
- Mark assumptions and placeholders clearly
- Refine only after direction confirmation

4. Provide options:
- Offer at least 2 materially different variants
- Vary tone/layout/interaction, not just colors

5. Keep quality intentional:
- Avoid generic AI-looking patterns
- Prioritize hierarchy, typography, and spacing over decoration

## Review Mode

When asked to review design quality, report findings first:
- Hierarchy
- Consistency
- Detail execution
- Interaction clarity
- Distinctiveness

Then give concrete fixes.
